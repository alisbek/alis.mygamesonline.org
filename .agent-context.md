## INPOST SHIPX API DETAILS

- Create shipment: `POST /v1/organizations/{org_id}/shipments` — async, status starts `created`
- Auth: `Authorization: Bearer <token>` — static token, no OAuth
- Parcel sizes: `small` (A: 80x380x640mm), `medium` (B: 190x380x640mm), `large` (C: 410x380x640mm) — max 25kg
- Labels: `GET /v1/shipments/{id}/label?format=pdf&type=A6` — only after `confirmed` status
- Dispatch: `POST /v1/organizations/{org_id}/dispatch_orders` — schedules courier pickup
- Geowidget v5: `<inpost-geowidget>` custom element, `config="parcelCollect"`, `onpoint` event -> `point.name` = locker ID
- Webhooks: configured in InPost Manager UI, respond HTTP 200 to GET, sent from IP `91.216.25.0/24`
- InPost Pay: Phase 2 LATER — requires signed contract
